Smart Restaurant Management System
This project is a console-based restaurant management system built in C for our Data Structure Lab course. The idea was simple: take what we learned about linked lists and turn it into something practical and fun. Restaurants felt like the perfect example because menus, orders, and reviews are all dynamic — they change constantly, just like nodes in a list.
The system has two roles:  
*Customer – browse the menu, place orders, request new items, leave reviews, and check out special offers.  
*Admin – manage the menu, update stock, read customer feedback, and review food requests (protected by password login).  
Behind the scenes, we used four linked lists to keep everything organized: one for menu items, one for orders, one for reviews, and one for customer requests. We also added features like automatic discounts for big orders, real-time stock tracking, and a clean memory deallocation process so nothing leaks when the program ends.
This project was our way of bridging theory with practice. It taught us how data structures can solve real-world problems while also giving us a taste of teamwork and software design. Future improvements could include persistent storage, a GUI, or even multi-user support — but for now, it’s a solid, working demo we’re proud of.
